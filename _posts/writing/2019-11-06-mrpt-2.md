MRPT 공부를 시작하기에 앞서, 대략적인 계획을 세워봤다. 다음과 같이 네단계로 나누어 진행하고자 한다.

1. **MRPT 라이브러리 구조 파악**
	- **MRPT Tutorial**[^1] 에서 설명하는 MRPT 프로젝트의 특징 및 라이브러리 구성이 어떻게 되어 있는지에 대해서 파악 
3. **ROS & MRPT 환경 구축**
	- ROS 에 연동해서 사용할 생각이기 때문에, ROS 와 MRPT를 함께 사용하는 세팅을 먼저 구축
4. **Simple Test 수행**
	- MRPT 라이브러리 핵심 모듈들(1단계에서 파악한 내용)을 하나 씩 직접 다뤄보기
	- 세팅된 ROS노드에서 활용해보기
5. **MRPT SLAM 예제중 하나를 분석**
	- 각 모듈들이 어떻게 사용되는지 보기

> Visualization 관련 모듈들은 ROS Rviz로 대체하고자 하므로 다루지 않을 생각이다. 

# 1. MRPT 라이브러리 구조
## 1. Tree overview
MRPT 라이브러리 소스는 다음과 같이 구성된다.
- **MRPT**
	- **apps** : *Applications*
	- **libs** : *C++ libraries*
		- **module A**
		- **module B**
		- ...

우리가 사용할 라이브러리는 **libs** 폴더에 있다. 이 폴더 하위에는**module** 들이 있고, 각 모듈은 아래에 설명된 공통된 구조를 갖는다.

MRPT에서 제공하는 **module**은 slam, nav, math, map ... 등이 있다. 자세한 구성은 [Full list of modules][fu] 

## 2. (Module) Name
어떤 모듈의 이름을 `<name>` 이라고 했을 때, 소스트리는 다음과 같이 구성되어 있다.

- **MRPT**
	- **apps** : *Applications*
	- **libs** : *C++ libraries*
		- `<name>`
			- **src**
				- precomp_hdr.cpp
				- registerAllClasses.cpp
				- (rest source files)
			- **include**
				- **mrpt**
					- `<name>`.h
					- `<name>`
						- link_pragmas.h
						- (rest include files)
			- **CMakeLists.txt**

이와같이 구성된 라이브러리를 Build하게 되면, `mrpt-<name>.dll`(Windows) 또는 `mrpt-<name>.so`(Unix) 파일이 생성된다.
