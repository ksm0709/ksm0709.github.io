# 준비
[MOLA의 공식 Documentation](https://docs.mola-slam.org/latest/building.html) 페이지를 참조하면, MOLA는 다음과 같은 의존성을 갖는다고 한다.( 중간 중간 우분투 16.04와 18.04의 설치 방법이 조금 다른 부분들이 있으니 주의가 필요하다.)

### (Build dependencies)[https://docs.mola-slam.org/latest/building.html]

-   A C++17 compiler. Either gcc-7, clang-4 or MSVC 2017 are good.
-   CMake >=3.9
-   Mandatory libraries:
    
    -   MRPT >= 1.9.9
    -   GTSAM >= 4.0.2
    -   yaml-cpp (will use an embedded version if not found in the system)
    
-   Optional libraries:
    
    -   opencv
    -   ros1: roscpp, sensor-msgs

### Boost, yaml-cpp 등 기타 라이브러리 설치

```bash
sudo apt-get install libboost-all-dev
sudo apt-get install libyaml-cpp-dev
sudo apt-get install libtbb-dev
sudo apt-get install libroscpp-dev libsensor-msgs-dev
```

### C++17 컴파일러 설치 (Ubuntu 16.04 에 해당)
> MRPT 설치시 이미 설치 했다면, 이 항목은 넘어가도 된다.

Ubuntu 18.04 의 경우에는 C++17을 기본으로 지원하지만, 16.04 버전은 그렇지 않다. 따라서, 다음과 같이 수동으로 설치해줘야 한다. [참조](https://gist.github.com/jlblancoc/99521194aba975286c80f93e47966dc5) 

```bash
# PPA 추가 후 설치
sudo apt-get install -y software-properties-common
sudo add-apt-repository ppa:ubuntu-toolchain-r/test
sudo apt update
sudo apt install g++-7 -y

# gcc, g++ 실행 파일에 심볼릭 링크 걸기
sudo update-alternatives --install /usr/bin/gcc gcc /usr/bin/gcc-7 60 \
                         --slave /usr/bin/g++ g++ /usr/bin/g++-7 
sudo update-alternatives --config gcc
gcc --version
g++ --version

# 선택 사항
# This one if you want the **all** toolchain programs (with the triplet names) to also point to gcc-7. 
# For example, this is needed if building Debian packages.
# If you are already are root (e.g. inside a docker image), remove the "sudo" below.
ls -la /usr/bin/ | grep -oP "[\S]*(gcc|g\+\+)(-[a-z]+)*[\s]" | xargs sudo bash -c 'for link in ${@:1}; do ln -s -f "/usr/bin/${link}-${0}" "/usr/bin/${link}"; done' 7
```

### CMake 설치 
MOLA 와 하위 의존성 패키지들이 요구하는 CMake의 최소 버전은 3.9 이상 이라고 한다. 아마 Blanco 교수님이 3.9에서 Build 한것 같으니 나도 3.9 버전을 찾아서 설치 했다. [참고 : CMake 공식 설치법](https://cmake.org/install/](https://cmake.org/install/)  

```bash
wget https://github.com/Kitware/CMake/releases/download/v3.9.4/cmake-3.9.4.tar.gz

# 압축 해제
tar -xvzf cmake-3.9.4.tar.gz
cd cmake-3.9.4

# Make & 설치
./bootstrap
make
sudo make install
```

### MRPT 설치
MRPT는 Mobile Robot Programming Toolkit의 약자로, MOLA를 개발한 Blanco 교수님이 만든 라이브러리 이다. MOLA의 많은 기능들이 MRPT 기반으로 만들어졌다. MOLA에서는 MRPT 1.9.9 버전 이상을 요구한다.  다음 명령어로 MRPT 설치를 진행한다. 
```bash
sudo add-apt-repository ppa:joseluisblancoc/mrpt
sudo apt update
sudo apt install libmrpt-dev mrpt-apps
```

### GTSAM 설치 
 GTSAM의 경우도 Ubuntu 16.04 와 18.04의 설치법이 다르다. 
 
 - **Ubuntu 18.04**
 우분투 18.04의 경우, Balnco 교수님이 빌드해놓은 버전을 다운로드 받으면 된다. 

```bash
 # GTSAM >=4.0.0, from this PPA, or build it from sources:
sudo add-apt-repository ppa:joseluisblancoc/gtsam-develop
sudo apt update
sudo apt install libgtsam-dev
```

- **Ubuntu 16.04**
우분투 16.04의 경우, 앞서와 같이 간단히 설치할 수 없고, 소스를 다운로드 받아 직접 빌드 해야한다.

```bash
# 기존에 설치된 gtsam이 있다면 제거
sudo apt remove libgtsam-dev

# gtsam 압축파일 다운로드, 압축 해제
wget https://github.com/borglab/gtsam/archive/4.0.2.tar.gz
tar -xvzf gtsam-4.0.2.tar.gz

# 빌드 수행
cd gtsam-4.0.2
mkdir build
cd build
cmake ..
sudo make install
```

### MOLA 빌드
드디어 MOLA 를 빌드할 차례이다. 이 때 주의할점이 하나 있는데, ROS가 설치되어 있는경우 ROS 라이브러리와 MOLA 에서 읽어오는 라이브러리간 충돌(?)이 발생한다는 점이다. (충돌이라기 보다는 CMake 에서 같은 이름의 라이브러리가 여러개 읽히고, 무엇을 읽을지 불명확 해서 생기는 문제)

따라서, 잠시 ROS 라이브러리를 읽을 수 없도록 조치한다.
```bash
vi ~/.bashrc
/source /opt/ros
```

```bash
git clone --recurse-submodules https://github.com/MOLAorg/mola.git

mkdir build
cd build

# 우분투 버전에 따라 다음 둘중 하나를 선택해서 수행
# (1) Ubuntu 18.04 >=
cmake ..
# (2) Ubuntu 16.04
cmake -DMOLA_BUILD_MARCH_NATIVE=OFF ..

# 최종 빌드
cmake --build .

# PATH 변수에 MOLA 바이너리 폴더 추가
echo 'set $PATH=$PATH:$HOME/code/mola/build/bin' >> ~/.bashrc
```
