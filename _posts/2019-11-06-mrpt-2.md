# 시작
MRPT 공부를 시작하기에 앞서, 대략적인 계획을 세워봤다. 다음과 같이 네단계로 나누어 진행하고자 한다.

1. **MRPT 라이브러리 구조 파악**
	- **MRPT Tutorial**[^tutorial] 에서 설명하는 MRPT 프로젝트의 특징 및 라이브러리 구성이 어떻게 되어 있는지에 대해서 파악 
2. **ROS & MRPT 환경 구축**
	- ROS 에 연동해서 사용할 생각이기 때문에, ROS 와 MRPT를 함께 사용하는 세팅을 먼저 구축
		- ROS1 Catkin 환경에서 MRPT 라이브러리를 링크하고 빌드 할 수 있도록 세팅
		- mrpt_navigation[^mrpt_nav] , mrpt_slam[^mrpt_slam] 이라는 ROS 패키지에서 어떻게 MRPT와 ROS를 연동하는지( Direct / Interface Method ) 확인하고 내 프로젝트에 적용 
3. **Simple Test 수행**
	- MRPT Samples[^samples] 에서 제공하는 샘플 활용
	- MRPT 라이브러리 핵심 모듈들(1단계에서 파악한 내용)을 하나 씩 직접 다뤄보기
	- 세팅된 ROS노드에서 활용해보기
	- 라이브러리 코드도 필요하다면 뜯어서 분석
	- 라이브러리와 관련된 수학적 배경에 대해서도 다루고자 함
4. **MRPT Application 예제중 하나를 분석**
	- 각 모듈들이 어떻게 사용되는지 보기

> Visualization 관련 모듈들은 ROS Rviz로 대체하거나 있는 App들을 활용만 하고자 하므로 다루지 않을 생각이다. 

# MRPT 라이브러리 구조
직접 MRPT 라이브러리 코드를 작성 할 일은 없겠지만, 코드를 이해하고 활용하기 위해서 어떤 형태로 라이브러리가 구성되어 있는지 알아야 할 필요가 있다. 아래의 내용은 튜토리얼 : MRPT libraries tree layout[^tree] 을 정리한 것이다.

## 1. Tree overview
MRPT 라이브러리 소스는 다음과 같이 구성된다.
- **MRPT**
	- **apps** : *Applications*
	- **libs** : *C++ libraries*
		- **module A**
		- **module B**
		- ...

우리가 사용할 라이브러리는 **libs** 폴더에 있다. 이 폴더 하위에는**module** 들이 있고, 각 모듈은 아래에 설명된 공통된 구조를 갖는다.

MRPT에서 제공하는 **module**은 slam, nav, math, map ... 등이 있다. 구체적으로는 [Full list of modules](#modules-full-list
) 를 보면 어떤 모듈들이 있는지 알 수 있다.

## 2. (Module) Name & Tree outline
어떤 모듈의 이름을 `<name>` 이라고 했을 때, 소스트리는 다음과 같이 구성되어 있다.

- **MRPT**
	- **apps** : *Applications*
	- **libs** : *C++ libraries*
		- `<name>`
			- **src**
				- precomp_hdr.cpp
				- registerAllClasses.cpp
				- (rest source files)
			- **include**
				- **mrpt**
					- `<name>`.h
					- `<name>`
						- link_pragmas.h
						- (rest include files)
			- **CMakeLists.txt**

이와같이 구성된 라이브러리를 Build하게 되면, `mrpt-<name>.dll`(Windows) 또는 `mrpt-<name>.so`(Unix) 파일이 생성된다.

| file | description |
|--|--|
| precomp_hdr.cpp | 이 파일은 오직 #include <mrpt/`<name>`.h> 만 포함한다 |
| registerAllClasses.cpp | **(optional)** CSerializable 객체들을 등록하는 역할. de-serialize 수행시에 객체 타입을 명시하지 않아도 되도록 하기위해 사용함. |
| `<name>`.h | Precompiled Header 이다. include/mrpt/`<name>` 하위의 헤더파일들을 include 한다. |
| link_pragmas.h | 1.5 버전에서 이 파일은 윈도우 DLL 빌드를 위해 사용되는 전처리 구문을 담고 있으나, 2.0 버전에서는 사용하지 않는것으로 보인다. |

## 3. The CMakeLists.txt  file
CMakeLists.txt 는 매우 간단하게 구성되는데, 앞서 정의했던 모듈의 이름 `<name>` 에 의존성을 갖는 다른 mrpt 모듈이나 외부 라이브러리를 연결해주면 끝이다. 
```bash
#---------------------------------------------
# Macro declared in "DeclareMRPTLib.cmake":
#---------------------------------------------
define_mrpt_lib(
# Lib name
name
# Dependencies
mrpt-base
....
)
```
> 이 CMakeLists.txt를 이용해서 빌드하게 되면, *_LIN.cpp 파일과 *_WIN.cpp 파일은 각 플랫폼에 따라 무시하는 기능이 있다. 이런 advanced 된 설정을 확인하려면 DeclareMRPTLib.cmake 를 참조하면 된다.
 
## 4. Unit testing
MRPT의 CMakeLists.txt는 **src** 폴더 아래의`*_unittest.cpp` 파일을 Unit test 용으로 인식한다.
Google의 gtest 라이브러리를 사용하며, `make test` 명령으로 빌드하면, 자동으로 `*_unittest.cpp` 파일을 인지해서 `test-<name>` 라이브러리로 빌드해 준다.

다음은 `*_unittest.cpp` 파일에 대한 예제이다.
```cpp
#include <mrpt/slam.h>
#include <gtest/gtest.h>

using  namespace  mrpt;
using  namespace  mrpt::utils;
using  namespace  std;

TEST(MyTestCaseName,  ThisTestName)
{
	double  err;
	// Do whatever...
	EXPECT_EQ(0,  err  )  <<  "Differences were found when checking ...\n";
}
```
